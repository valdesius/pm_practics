# Фаза 1 — Выбор концепции проекта

## 1. Альтернативные концепции

**A1. Telegram-бот “VasyokLibraryBot” (книги)**
- Добавление книг по ISBN (текст/скан/фото), ручной ввод.
- Хранение в БД, поиск по библиотеке, статусы (хочу/читаю/прочитал).
- Проверка дубликатов, базовые рекомендации.
- Интеграция с провайдерами метаданных (RU/EN).

**A2. Веб-приложение (PWA) “Моя библиотека”**
- Веб-кабинет: коллекция, теги, фильтры, импорт/экспорт.
- Авторизация, синхронизация, простые рекомендации.
- Акцент на UI/UX и веб-архитектуру.

**A3. Мобильное приложение “Сканер книг”**
- Камера → ISBN/обложка → распознавание → добавление в коллекцию.
- Оффлайн-режим, синхронизация, уведомления.
- Самая “продуктовая” версия, но дороже по разработке.

**A4. Браузерное расширение “Книжный помощник”**
- На страницах магазинов/каталогов показывает: “есть ли у меня”, цена/альтернативы.
- Список желаемого, быстрые заметки.
- Интеграции и ограничения браузерных API.

## 2. Критерии выбора

Шкала оценок альтернатив по критериям: **1..10** (больше — лучше).

Критерии (все заданы как выигрышные *оценки*; для C2 и C3 это именно “оценка удобства/простоты”, а не “объективная сложность”, поэтому больше — лучше):

- **C1 Польза/спрос**: насколько решение нужно пользователю, потенциальная ценность.
- **C2 Реализуемость за семестр**: насколько реалистично сделать MVP (10–15 use cases) силами команды.
- **C3 Простота данных/интеграций**: доступность API/парсинга, риски блокировок, сложность интеграций (чем проще — тем выше оценка).
- **C4 Богатство для оценок по предмету**: насколько удобно делать дальнейшие оценки (COSMIC/UCP/экспертные оценки/планирование).

---

## 3. Эксперты и формат оценок

Экспертная группа (3 участника):
- **E1**, **E2**, **E3**.

Каждая оценка задаётся как трёхточечная:
- **P** (пессимистичная), **E** (ожидаемая), **O** (оптимистичная), где P ≤ E ≤ O.

Трёхточечная оценка трактуется как **треугольное нечёткое число (P, E, O)** для Fuzzy TOPSIS.
