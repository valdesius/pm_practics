# Постановка задачи распределения (оптимизация)

Требования из задания:
- разбить работы Фазы 3 на задачи
- оценить трудозатраты задач экспертно
- дать индивидуальные предпочтения задач
- сформулировать постановку (целевая функция + ограничения)
- решить любым инструментом (OpenSolver/NEOS/Pyomo и т.д.) (см. Задание, Фаза 2)

## Переменные

x[t,m] ∈ {0,1}
- x[t,m] = 1, если задача t назначена участнику m

## Данные

- h[t] — ожидаемые трудозатраты задачи (PERT, часы)
- p[t,m] — предпочтительность задачи t для участника m (1..10)
- cap[m] — доступные часы участника m на выполнение Фазы 3

## Ограничения

1) Каждая задача назначена ровно одному участнику:  
   ∑_m x[t,m] = 1  для всех t

2) Нагрузка участника не превышает доступное время:  
   ∑_t h[t] * x[t,m] ≤ cap[m]  для всех m

## Целевая функция

Максимизируем суммарную удовлетворённость распределением, но добавляем мягкое требование “не перегружать одного человека”.

Вариант (практичный):  
max  ∑_t ∑_m p[t,m] * x[t,m]  −  λ * ∑_m |Load[m] − AvgLoad|

где:
- Load[m] = ∑_t h[t] * x[t,m]
- AvgLoad = (∑_t h[t]) / 3
- λ — вес штрафа за дисбаланс (например 0.6)

Почему так:
- первая часть делает распределение “по компетенциям/интересу”
- штраф выравнивает нагрузку, чтобы не было перекоса

# Решение задачи распределения

Инструмент решения:
- так как задач всего 12, решение найдено перебором всех вариантов 3^12 с проверкой ограничений по cap[m] и выбором максимума целевой функции.

Ограничения по времени (примерные, на Фазу 3):
- cap[Малышев Владислав] = 15 ч
- cap[Капустин Максим] = 14 ч
- cap[Ульянов Николай] = 14 ч

Параметр баланса:
- λ = 0.6

## Лучшее найденное распределение

**Капустин Максим:**
- T1 (митинг)
- T2 (акторы + UC-шаблон)
- T3 (UC1–UC6)
- T11 (оформление репозитория)
- T12 (финальная вычитка + митинг)

**Малышев Владислав:**
- T5 (COSMIC UC1–UC6)
- T6 (COSMIC UC7–UC13 + проверка)
- T8 (UCP: TCF/ECF + расчёт)
- T10 (свод, выводы)

**Ульянов Николай:**
- T4 (UC7–UC13)
- T7 (UCP: акторы + классификация UC)
- T9 (экспертные оценки + PERT)

## Нагрузки (PERT-часы)

- Load[Малышев Владислав] = 14.08 ч
- Load[Капустин Максим] = 13.25 ч
- Load[Ульянов Николай] = 12.17 ч

Средняя нагрузка = 13.17 ч, перекос небольшой — решение сбалансировано.

## Почему это разумно

- Капустин Максим закрывает “много текста/структуры”: UC-описания + оформление + финальная вычитка.
- Малышев Владислав берёт “аналитику/метрики”: COSMIC и расчётную часть UCP + финальные выводы.
- Ульянов Николай выступает “мостом”: вторая половина UC + часть UCP и сбор экспертных оценок.
